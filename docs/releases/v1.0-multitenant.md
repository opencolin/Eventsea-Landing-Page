# v1.0 — Multi-tenant prod app + first paying customer

> Status: **planned**.
> Branch: `release/v1.0-multitenant`
> Estimated effort: 4–6 weeks for 2 engineers
> Depends on: v0.3 with at least 2 paying design partners

## Goal

Promote Eventsea from a landing page + audit pipeline + admin tool into a real product with auth, multi-tenant orgs, and a usable web app for the radar UI. Mirror what's already live at `event-radar-mvp.phantastic.ai`, but inside this codebase.

Success metric: **≥10 paying customers active in the past 14 days. ≥3 of them generated by self-serve signup (no founder onboarding call).**

## Non-goals

- Not yet shipping team workspaces with sharing/permissions (v1.5).
- Not yet shipping sponsor matchmaking transactions (v2.0).
- Not yet shipping the venue booking flow (v2.0).
- Not yet replacing Luma — Eventsea is the *layer above*, not a Luma competitor.

## In-scope features

### 1. Auth + orgs
- Clerk or Auth.js (pick one — Clerk recommended for solo-founder ops).
- `organizations` table; users belong to orgs with roles (`owner`, `admin`, `member`).
- Sign in with Google + Email.

### 2. Self-serve calendar import
- Org owner pastes Luma calendar URL(s) on first run.
- Eventsea polls those calendars on a daily schedule, surfaces new events automatically.

### 3. The radar UI (port from prototype)
- Mirror the screen the founder shared (event radar mock):
  - Ranked events list with score, P1/P2/P3 chip, sponsors/hosts strip
  - Semantic filter sliders + custom filter input
  - Saved views per user
  - CSV export
- Implementation: `client/src/pages/app/radar.tsx` + sub-components.
- Hybrid search powered by LanceDB or pgvector — pick pgvector if Neon is the DB (simpler).

### 4. Billing
- Stripe integration. Three tiers as in `/pricing`:
  - Community: free
  - Pro: $X/mo per seat (validate price in v0.3 with design partners)
  - Scale: usage- or contract-based
- Self-serve upgrade flow.

### 5. Onboarding flow
- Step 1: paste 1+ Luma calendars (or upload CSV).
- Step 2: pick or build your ICP filter set.
- Step 3: see the first ranked feed.
- Day 1 send: welcome email with weekly-digest sample.

## Risks + mitigations

- **Scope creep** — v1.0 is intentionally a port of an existing prototype. Don't redesign.
- **Pricing wrong** — exit v1.0 early if churn is high; revisit pricing with the design partners from v0.3.

## Acceptance test

A new user signs up at `eventsea.ai`, lands in onboarding, pastes a Luma URL, configures one filter set, and sees ranked events in <5 minutes from sign-up. They upgrade to Pro on the same day.
