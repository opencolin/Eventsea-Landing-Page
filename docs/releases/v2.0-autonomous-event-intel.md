## Goal

The app finds and processes events on its own. No human in the loop:
- **Tavily** (search/extract/crawl) discovers event calendars and registration/attendee data across the web.
- **Token Factory** (open-model inference) does screening, classification, and scoring at API cost rather than premium-model cost.
- **Scheduled job** rebuilds and redeploys the dashboard automatically.

The deployed page becomes self-updating — events in, ranked leads out.

Success metric: **at least 50 events autonomously discovered, ingested, and scored per week. ≥80% of "top events" lists generated without human review touch.**

## Non-goals

- Not replacing the v0.2 CSV-upload flow. Self-serve uploads remain for sponsors who don't want autonomous discovery.
- Not yet supporting non-Luma event sources beyond what Tavily discovers (Eventbrite, Meetup, conference websites). Add per market demand.

## In-scope features

### 1. Tavily-powered event discovery
- Configurable query templates: "{topic} hackathons 2026", "{topic} demo days", "{topic} meetups in {region}".
- Scheduled Tavily search + crawl per configured ICP.
- Dedupe across runs by event URL + start date.

### 2. Tavily extract for guest data
- For each discovered event, attempt to extract: title, date, location, host(s), sponsor(s), speaker(s), guest count, RSVP URL.
- If a public guest list is available (some Luma calendars expose it), extract attendee handles.
- If only aggregate metadata (no guest list), score the event itself for sponsor fit; flag for later guest-list ingestion via v1.0 Luma API path.

### 3. Token Factory inference layer
- Replace the Claude-API screening path with Nebius Token Factory open-model inference for the **light screening** workload (high volume, low complexity). Keep Claude Sonnet/Opus for deep enrichment (low volume, high complexity).
- Cost target: ≤$0.005 per screened lead.
- Quality bar: V/P/U classification accuracy within 5% of Claude Haiku baseline.

### 4. Self-rebuilding dashboard
- A scheduled Inngest job kicks off every 6 hours: re-ingest discovered events, re-score, regenerate the dashboard HTML, redeploy.
- Versioned URLs so sponsors can see "as of {timestamp}" snapshots.

### 5. Trust + safety
- Tavily-discovered claims (sponsor lists, attendee handles, speaker bios) go through the same skill-prescribed verification rules: LinkedIn URL/slug is the identity anchor; "not found" is the honest answer.
- "Adversarially re-verify before publishing" applies to autonomous output too — gate by confidence threshold; below that, flag for human review.

## Files / endpoints

- `server/discovery/tavily.ts` — search + extract + crawl orchestration
- `server/inference/token-factory.ts` — Nebius Token Factory client
- `server/jobs/autonomous-rebuild.ts` — Inngest cron, all-the-way pipeline
- `server/discovery/queue.ts` — discovered-events queue with dedup
- `shared/schema.ts` — `discovered_events`, `discovery_runs`, `inference_runs`

## Risks + mitigations

- **Hallucinations cascade in autonomous mode.** Mitigation: per-claim confidence scoring; below threshold → flag for human review; never publish unverified "competitor" labels or funding figures (per skill rule).
- **Tavily rate limits / cost.** Mitigation: cache aggressively per event URL; ICP-prioritize discovery; weekly cost cap with circuit breaker.
- **Token Factory quality drift.** Mitigation: weekly A/B sample of N=50 against Claude baseline; pull TF if quality drops below threshold.

## Acceptance test

A sponsor configures their ICP. Within 24 hours, the autonomous pipeline discovers ≥5 relevant events from across the web (not Luma-connected by the sponsor), extracts metadata, scores them, surfaces a "top 3 events to sponsor this quarter" recommendation. Zero operator touch from event discovery to dashboard publish.
