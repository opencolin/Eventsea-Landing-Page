## Goal

Replace manual CSV exports with the Luma guest list API. Eventsea pulls registrations on a schedule, diffs new approvals automatically (skill Step 7), and re-screens only additions — so sponsors stop having to upload a new CSV every few days during a multi-week registration window.

Success metric: **90% of v0.2 customers connect at least one Luma calendar in the first 14 days of v1.0 launch. Re-screening latency on a new RSVP under 10 minutes.**

## Non-goals

- Not yet replacing Luma as the RSVP layer. Eventsea reads Luma; Eventsea does not host RSVPs.
- Not yet adding Eventbrite or Meetup ingestion. Add when ≥3 customers ask.

## In-scope features

### 1. Luma OAuth + API key
- Sponsor connects their Luma account to Eventsea (host-side OAuth or API key paste, depending on what Luma allows).
- Eventsea stores credentials encrypted (per-tenant key in Vault or KMS).

### 2. Guest list polling
- For each connected Luma calendar, poll the guest list endpoint on a 1-hour schedule (configurable).
- Detect new approvals, new RSVPs, and removals.
- Persist the delta as job inputs to the v0.2 pipeline.

### 3. Auto-diff re-screening
- New approvals → run through the full Step 2 light screening (or deep if high signal).
- Removals → drop from leads; if a curated slot was held, surface the gap and propose a backfill.
- All updates → recompute Top-N / judges + rebuild dashboard.

### 4. Webhook fallback
- If Luma supports webhooks (event-driven instead of polling), prefer that.
- Polling is the fallback for latency.

## Files / endpoints

- `server/integrations/luma.ts` — auth handshake + guest list fetcher
- `server/jobs/luma-poll.ts` — Inngest scheduled function (cron 0 * * * *)
- `server/jobs/luma-webhook.ts` — Inngest event-handler for Luma webhooks (if available)
- `client/src/pages/app/integrations.tsx` — connect-luma UI per sponsor org
- `shared/schema.ts` — `luma_connections`, `luma_calendars`, `luma_sync_runs`

## Acceptance test

A v0.2 sponsor connects their Luma host account. Within 1 hour, the dashboard shows their connected calendar's current guest list. They watch one new RSVP come in; within 10 minutes the new lead appears on the dashboard with V/P/U verdict + score columns + influence rank.
