# v2.0 — Marketplace transactions: sponsor matching + venue booking

> Status: **planned**.
> Branch: `release/v2.0-marketplace-tx`
> Estimated effort: 8–12 weeks for 3 engineers
> Depends on: v1.5 with a recurring user base and named sponsor + venue customers

## Goal

Until v2.0, Eventsea has been an *information platform*. v2.0 is the **transactional marketplace** described in the original goal: organizers list events, sponsors fund them through Eventsea, venues book through Eventsea, with money moving end-to-end.

Success metric: **$1M GMV in the first 6 months post-launch. ≥30 transactions/month. ≥10 venues with calendars at 50%+ utilization through Eventsea.**

## Non-goals

- Not building event ticketing (Luma stays the ticketing layer).
- Not building messaging beyond structured intros + inquiry threads. Slack/Email is the messaging layer for now.
- Not building advanced ML matching. Embedding similarity + filter rules is enough.

## In-scope features

### 1. Sponsor marketplace
- Sponsor org creates a sponsor program: ICP, target geos, budget per event, tier preferences (title / track / lunch / API credits / etc.).
- Eventsea matches the sponsor to listed events whose audience fits.
- Sponsor can express interest with a one-click "sponsor this" → opens a structured intro thread with the organizer.
- Stripe Connect handles payment if the deal closes through Eventsea (with a take rate, e.g. 5%).
- Sponsor sees an ROI dashboard per sponsored event: people reached, submissions on their stack, credits consumed, post-event audience list.

### 2. Venue marketplace
- Venue org creates a listing: capacity by layout, AV, Wi-Fi, catering, rate range, available dates.
- Eventsea matches venues to listed events.
- Organizer sends a structured inquiry; venue responds with availability + quote.
- Booking confirmed → calendar block + invoice via Stripe Connect.

### 3. Organizer payouts + escrow
- Organizers receive sponsor money via Stripe Connect (Eventsea takes a fee).
- Optional escrow: sponsor pays at signing, funds release at event completion.
- 1099-style year-end reporting (US only initially).

### 4. Sponsor credit distribution
- Sponsor uploads credit packs (API keys, promo codes, vouchers).
- Eventsea distributes one credit pack per accepted attendee, tracks redemption, reports back to sponsor.

### 5. Public marketplace browsing
- `/marketplace/events` becomes a real public surface: filterable, paginated, RSS feed.
- Each event card shows sponsorship tiers available, venue assignment, RSVP link out to Luma.

## Risks + mitigations

- **Marketplace cold-start** — solve by seeding both sides with hand-onboarded design partners from v0.3.
- **Trust + payments** — Stripe Connect is heavy. Allocate 2 weeks just for onboarding flow + KYC.
- **Take-rate friction** — start at 5%, A/B against organizer satisfaction. Have a sub-10% promise as anchor.

## Acceptance test

End-to-end demo: An organizer lists "ETHDenver Side Event" with a $5k sponsor ask + venue need. A sponsor and a venue both express interest within 7 days through the marketplace. Money moves through Stripe Connect. Eventsea takes its cut. Post-event, the sponsor sees an ROI report with attendee list (opted in).
